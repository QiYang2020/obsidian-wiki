# Technical Documentation

技术文档的结构化写作方法，涵盖技术交底书、操作手册、流程图等常见文档类型。

## 技术交底书构成模块

技术交底书是专利申请前的核心文档，需包含以下模块：

1. **技术领域** — 所属技术领域概述
2. **背景技术** — 现有技术的不足与痛点
3. **发明内容** — 技术方案的整体描述
4. **技术方案** — 具体实现方式与创新点
5. **有益效果** — 相比现有技术的优势
6. **附图说明** — 图纸编号与说明
7. **具体实施方式** — 详细实施例

写新交底书时，按此结构组织内容即可。

## 操作手册结构整理

操作手册的写作原则：

- **按实际操作顺序排列**，不要过分延伸
- 根据自然语言描述调整层级结构
- 话题归属清晰，避免交叉

**示例：打印及烫画操作手册**

按设备/流程分节，每节包含：
- 准备工作
- 操作步骤（编号列表）
- 注意事项

**示例：杯子热转印机操作手册**

单独设备的操作手册，仅根据实际操作描述整理，不做过度扩展。

## 流程图大纲生成

根据业务信息生成流程图大纲的方法：

1. 提取关键业务节点
2. 确定节点间的关系（顺序/并行/条件分支）
3. 用标准流程图符号组织
4. 标注决策条件与分支走向

流程图大纲可用于：
- 技术方案说明
- 操作流程可视化
- 项目管理与评审
